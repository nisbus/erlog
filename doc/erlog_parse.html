Description

Parses Erlog tokens into Erlog terms. Based on the Standard prolog parser and directly coded from the parser description. To handle back-tracking in the parser we use a continuation style using funs where each fun handles one step of what follows. This allows back-tracking. This may not be a specially efficient way of parsing but it is simple and easy to derive from the description. No logical variables are necessary here.
